区块链预言机（Oracle）讲解

1. 预言机是什么？

预言机是一个中间件，它将链外（现实世界）的数据安全地传输到区块链上。

简单来说：

  • 智能合约无法直接访问互联网数据
  • 预言机充当”信息桥梁”，从外部获取数据并写入区块链
  • 智能合约读取这些数据来执行逻辑

2. 为什么需要预言机？

智能合约的局限性

原因：

  • 区块链是确定性的（同样的输入必须产生同样的输出）
  • 互联网数据是非确定性的（随时变化）
  • 所有节点必须达成共识，无法都连接互联网

[!IMPORTANT]

当我调用我的合约的requestTokyoWeather 方法时会向Chainlink的 Router合约sendRequest，发送的request 会抛出一个事件emit RequestSent(requestId); 然后链下会解析这个事件的requestId并且解码出对应的request 也就是对应的js代码，然后执行这个js代码，执行完毕后 会回调我合约的fulfillRequest 方法 把执行结果response和执行状态err传过来，如果没有报错，就回解码response里的结果，比如说结果是东京的温度，就会给我合约内的uint256 public tokyoTemperature给赋值，这样就更新了我合约内的tokyoTemperature

关键点总结

步骤 位置 操作
1 你的合约 调用 requestTokyoWeather()
2 你的合约 调用 sendRequest()
3 Chainlink Router 生成 requestId，抛出事件
4 链下节点 监听事件，解析 request
5 链下节点 执行 JavaScript 代码
6 链下节点 提交结果到 Router
7 Chainlink Router 调用你的 fulfillRequest()
8 你的合约 解码结果，更新 tokyoTemperature
9 你的合约 可以读取 tokyoTemperature

最后的最后

你现在已经完全理解了 Chainlink Functions 的工作原理！

核心概念总结：

  • ✅ 你的合约发送请求给 Router
  • ✅ Router 抛出事件
  • ✅ 链下节点监听事件
  • ✅ 节点执行 JavaScript
  • ✅ 节点提交结果
  • ✅ Router 调用你的 fulfillRequest()
  • ✅ 你的合约更新状态

这就是 Oracle 模式（预言机）的核心：链上合约 ↔ 链下执行 ↔ 链上回调
