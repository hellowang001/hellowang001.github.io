Uniswap

UniswapV2Router02.sol路由合约

swapTokensForExactTokens函数

用户指定输入代币的精确数量，路由合约帮你按当前汇率和滑点计算能换到的最少输出代币数量，并完成兑换。

函数签名

主要流程

  1. 转入输入代币
    • 用户先通过 ERC20 approve 授权路由合约使用输入代币。
    • 路由合约调用 transferFrom 从用户地址转入 amountIn 输入代币到第一个交易对合约。
  2. 计算兑换路径中每一步的输出数量
    • 使用 UniswapV2Library.getAmountsOut 计算整个路径上每一步的输出金额。
    • 这个函数基于当前池子储备量和手续费计算每一步的输出。
  3. 检查输出数量是否满足用户要求
    • 确保最终输出代币数量 >= amountOutMin，防止滑点过大。
  4. 执行多步兑换
    • 路由合约调用每个交易对合约的 swap 方法，依次完成路径中所有交易对的兑换。
    • 最终输出代币发送到 to 地址。
  5. 返回每一步的代币数量数组
    • 返回 amounts，包含路径上每个代币的数量（输入和输出）。

关键点

  • 用户输入数量固定，输出数量根据池子状态动态计算。
  • 通过 amountOutMin 保护用户免受价格波动影响。
  • 支持多跳兑换（路径长度 > 2）。

2. swapTokensForExactTokens

函数作用

用户指定想要获得的输出代币数量，路由合约帮你计算完成兑换所需的最少输入代币数量，并完成兑换。

函数签名

主要流程

  1. 计算兑换路径中每一步的输入数量
    • 使用 UniswapV2Library.getAmountsIn 计算整个路径上每一步所需的输入金额。
    • 这个函数基于当前池子储备量和手续费计算输入数量。
  2. 检查输入数量是否满足用户要求
    • 确保最终输入代币数量 <= amountInMax，防止滑点导致输入过多。
  3. 转入输入代币
    • 路由合约调用 transferFrom 从用户地址转入计算好的输入代币数量到第一个交易对合约。
  4. 执行多步兑换
    • 路由合约调用每个交易对合约的 swap 方法，依次完成路径中所有交易对的兑换。
    • 最终输出代币发送到 to 地址。
  5. 返回每一步的代币数量数组
    • 返回 amounts，包含路径上每个代币的数量（输入和输出）。

关键点

  • 用户输出数量固定，输入数量根据池子状态动态计算。
  • 通过 amountInMax 保护用户免受价格波动影响。
  • 支持多跳兑换。

总结对比

函数名 输入固定还是输出固定 用户保护参数 计算方法 适用场景
swapExactTokensForTokens 输入数量固定 amountOutMin getAmountsOut 用户知道自己要卖多少代币，想获得尽可能多的输出
swapTokensForExactTokens 输出数量固定 amountInMax getAmountsIn 用户知道自己想买多少代币，愿意最多花多少钱

swapExactTokensForTokens 和 swapTokensForExactTokens 的返回值

这两个函数的返回值都是：

返回值 amounts 是什么？

  • amounts 是一个数组，包含了兑换路径（path）中每个代币的数量。
  • 数组长度等于 path.length。
  • 具体来说：
    • amounts[0] 是输入代币的数量（swapExactTokensForTokens 中就是用户输入的 amountIn，swapTokensForExactTokens 中是计算出的实际输入数量）。
    • amounts[i] 是路径中第 i 个代币的数量，中间经过每个交易对的兑换后递减或递增。
    • amounts[path.length - 1] 是最终输出代币的数量。

总结

这两个方法都会进行真正的swap操作，只是通过限制输入和输出的金额两种方式来调用而已

获取预期输入和输出金额

getAmountsIn、getAmountsOut、getAmountOut、getAmountIn

这几个方法都是一样的道理，都是获取对应的输入输出金额而已

  • getAmountOut：给定一项资产的输入量和对储量，返回另一项资产的最大输出量

    • amountIn：用户输入的代币数量（想要兑换的数量）
    • reserveIn：输入代币在池子中的储备量
    • reserveOut：输出代币在池子中的储备量

  • getAmountIn：给定一项资产的输出量和对储备，返回另一项资产所需的输入量
